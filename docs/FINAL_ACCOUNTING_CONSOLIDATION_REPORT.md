# تقرير توحيد النظام المالي والمحاسبي

تاريخ التنفيذ: 23 مايو 2026

## التحسينات المحاسبية

- تحويل صفحة `/accounting` من redirect إلى لوحة تحكم مالية فعلية.
- إضافة ملخص مالي مباشر:
  - رصيد الخزن.
  - وارد اليوم.
  - منصرف اليوم.
  - إيرادات الفترة.
  - المصروفات.
  - مديونيات العملاء.
  - مستحقات الموردين.
  - الفواتير المتأخرة.
- إعادة ترتيب قائمة المحاسبة لتصبح مركز مالي تشغيلي:
  - لوحة التحكم المالية.
  - الخزن.
  - حركات الخزنة.
  - التحصيلات.
  - المدفوعات.
  - تحويلات الخزن.
  - القيود اليومية.
  - دليل الحسابات.
  - الحسابات المدينة والدائنة.
  - التقارير المالية.

## ترقيات الخزنة

- إضافة حقول `beforeBalance` و`afterBalance` لكل حركة خزنة.
- تحديث خدمة الخزنة لتسجل الرصيد قبل وبعد كل وارد أو منصرف.
- إضافة مركز حركات خزنة عام:
  - `/accounting/treasury/transactions`
  - فلاتر حسب التاريخ والنوع والمرجع.
  - تصدير CSV.
  - عرض الخزنة، النوع، الاتجاه، المبلغ، الرصيد قبل وبعد، والمرجع.
- تحسين Dashboard الخزنة بإضافة:
  - وارد اليوم.
  - منصرف اليوم.
  - مديونيات قيد التحصيل.
  - مستحقات مطلوبة.
  - عدد الفواتير المتأخرة.

## تحويلات الخزن

- إضافة مسار جديد:
  - `/accounting/treasury/transfers`
- إضافة API:
  - `POST /api/cashboxes/transfers`
- التحويل يتم داخل `prisma.$transaction`:
  - حركة منصرف من خزنة المصدر.
  - حركة وارد إلى خزنة الوجهة.
  - قيد يومية متوازن مرتبط بالتحويل.
  - سجل audit للعملية.
- يتم منع:
  - التحويل لنفس الخزنة.
  - التحويل من خزنة غير نشطة.
  - التحويل إذا كان رصيد المصدر غير كاف.

## تحسينات القيود اليومية

- إضافة صفحة تفاصيل قيد:
  - `/accounting/journal-entries/[id]`
- صفحة التفاصيل تعرض:
  - رقم القيد.
  - التاريخ.
  - حالة الترحيل.
  - إجمالي المدين والدائن.
  - المرجع التشغيلي.
  - رابط للمستند المصدر عند توفره.
  - بنود القيد مع الحساب والبيان والمدين والدائن.
  - تاريخ الإنشاء والتحديث والترحيل وحالة العكس.
- ربط رقم القيد في جدول القيود بصفحة التفاصيل.

## الحسابات المدينة والدائنة

- تم ربطها داخل قائمة المحاسبة بالمسارات الحالية:
  - `/reports/receivables`
  - `/reports/payables`
- التقارير الحالية تعرض ملخصات وفلاتر وتصدير CSV وتحدث بياناتها بدون cache.

## التقارير المالية

- تم ربط قائمة الدخل والميزانية العمومية والتقارير المالية من قائمة المحاسبة.
- إضافة صفحة تدفقات نقدية:
  - `/accounting/cash-flow`
- صفحة التدفقات تستخدم API المحاسبة الحالي:
  - `/api/accounting/cash-flow`

## تكامل المصروفات مع الخزنة

- عند إنشاء مصروف يحتوي على `cashboxId`:
  - يتم تسجيل المصروف.
  - يتم إنشاء حركة خزنة منصرف داخل نفس transaction.
  - يتم ربط الحركة بالمرجع `Expense`.
- تم الحفاظ على منطق القيود المحاسبية الحالي للمصروفات بدون إعادة كتابة المعمارية.

## فحوصات النزاهة

- كل حركة خزنة جديدة تحفظ الرصيد قبل وبعد الحركة.
- تحويل الخزنة ينتج حركتين نقديتين وقيد يومية متوازن.
- المدفوعات والفواتير ما زالت تستخدم الخدمات canonical الموجودة.
- لم يتم تغيير منطق FIFO/WAC أو التصنيع أو المخزون.
