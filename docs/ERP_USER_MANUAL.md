# دليل مستخدم نظام ERP

## 1. نظرة عامة على النظام

نظام ERP متعدد المستأجرين يدمج المحاسبة والمخزون والمبيعات والمشتريات والتصنيع في منصة واحدة.

| الوحدة | الوظيفة |
|--------|---------|
| المحاسبة | قيد مزدوج، قيود يومية تلقائية، ميزانية، تدفقات نقدية |
| المخزون | حركات مخزون، تكلفة FIFO/WAC، تسويات ونقل |
| المبيعات | عملاء، فواتير مبيعات، مدفوعات، مرتجعات |
| المشتريات | موردون، فواتير مشتريات، استلام بضاعة |
| التصنيع | أوامر إنتاج، BOM، استهلاك مواد وإدخال منتج تام |
| المستودعات | مخازن متعددة، رصيد لكل منتج/مخزن |

**مبدأ التشغيل:** أي عملية تجارية (فاتورة، دفع، إنتاج) تمر عبر محرك التنفيذ المركزي — لا يُحدَّث المخزون أو الدفتر إلا عبر الخدمات المعتمدة.

---

## 2. أدوار المستخدمين

| الدور | الصلاحيات النموذجية |
|-------|---------------------|
| **Admin** | إعداد النظام، المستخدمون، كل الوحدات |
| **Accountant** | قيود، تقارير مالية، مدفوعات، إقفال فترات |
| **Warehouse Manager** | منتجات، مخازن، تسويات، نقل، جرد |
| **Sales User** | عملاء، فواتير مبيعات، عروض |
| **Purchase User** | موردون، فواتير مشتريات، طلبات شراء |

الصلاحيات تُفرض على مستوى API (`checkPermission`) ومسارات لوحة التحكم.

## 3. دليل الاستخدام خطوة بخطوة

### إنشاء منتج (منتج نهائي)

1. **المخزون → المنتجات → إضافة منتج**
2. أدخل الاسم، السعر، التكلفة، الوحدة، المخزن (اختياري).
3. **لا تُدخل الرمز يدوياً** — يُولَّد تلقائياً (مثال: `PRD-2026-00001`).
4. احفظ. إن وُجد رصيد افتتاحي، يُسجَّل كحركة تكلفة افتتاحية.

### إنشاء مادة خام

1. **المخزون → المواد الخام → إضافة**
2. نفس الخطوات؛ الرمز بصيغة `RAW-YYYY-XXXXX`.

### إنشاء عميل / مورد

1. **العملاء** أو **الموردون → إضافة**
2. الاسم العربي مطلوب؛ الرمز `CUS-…` أو `VEN-…` يُولَّد عند الحفظ.

### إنشاء مستودع

1. **المخزون → المستودعات → إضافة**
2. الرمز `WRH-YYYY-XXXXX` يُولَّد تلقائياً.

### فاتورة مبيعات

1. **الفواتير → مبيعات → فاتورة جديدة**
2. اختر العميل، التاريخ، البنود.
3. **مسودة:** لا خصم مخزون ولا قيود نهائية.
4. **حفظ/ترحيل:** يُخصم المخزون ويُنشأ القيد عبر محرك التنفيذ.
5. رقم الفاتورة `SAL-YYYY-XXXXX` يُولَّد من الخادم.

### فاتورة مشتريات

مماثلة للمبيعات؛ الرمز `PUR-…` والمورد من قائمة الموردين.

### المدفوعات

1. **المحاسبة → المدفوعات**
2. سجّل دفعة واربطها بفاتورة/عميل.
3. التخصيص لا يتجاوز المتبقي على الفاتورة.
4. عكس الدفع يعكس القيود المرتبطة.

### التصنيع

1. أنشئ أمر إنتاج من BOM.
2. عند **الإكمال:** استهلاك مواد خام ثم إدخال منتج تام مع قيد تكلفة.

## 4. الأتمتة

### نظام الرموز التلقائي

| الكيان | البادئة | مثال |
|--------|---------|------|
| عميل | CUS | CUS-2026-00001 |
| مورد | VEN | VEN-2026-00001 |
| منتج | PRD | PRD-2026-00001 |
| مادة خام | RAW | RAW-2026-00001 |
| مستودع | WRH | WRH-2026-00001 |
| فاتورة مبيعات | SAL | SAL-2026-00001 |
| فاتورة مشتريات | PUR | PUR-2026-00001 |

- التوليد في **الخادم فقط** (`allocateEntityCode`).
- الواجهة تعرض الرمز للقراءة فقط بعد الإنشاء.
- السجلات القديمة (SUP, FG, INV…) تبقى صالحة؛ التسلسل يحترم أعلى رقم موجود.

### التكلفة FIFO/WAC

تُحسب عند الصرف والإدخال عبر `inventory-costing` ولا تُعدَّل يدوياً من شاشة المنتج.

### حركات المخزون

كل زيادة/نقصان عبر `inventory-movement.service` مرتبط بمرجع (فاتورة، تسوية، إنتاج).

### القيود اليومية

تُنشأ عند ترحيل الفاتورة/الدفع/الإنتاج عبر خدمات التنفيذ مع **Posting Guard**.

### تخصيص المدفوعات

يُحدَّث رصيد العميل/المورد ويُنشأ قيد بنكي/صندوق حسب نوع الدفع.

## 5. سلوك النظام

### إنشاء فاتورة

```
مسودة → (اختياري) تعديل → ترحيل → مخزون + قيد + رصيد طرف
```

### ترحيل دفعة

```
دفعة → تخصيص → قيد → تحديث حالة الفاتورة (مدفوعة/جزئية)
```

### حذف فاتورة

```
تحقق صلاحيات → عكس مدفوعات → عكس مخزون وتكلفة → عكس قيود → حذف/إلغاء
```

### التصنيع

```
أمر → صرف خامات → إكمال → إدخال تام + قيد تكلفة إنتاج
```

## 6. معالجة الأخطاء

| الخطأ | المعنى | الإجراء |
|-------|--------|---------|
| مخزون غير كافٍ | الكمية المطلوبة أكبر من المتاح | قلّل الكمية أو استلم مشتريات أولاً |
| تجاوز الدفع | المبلغ أكبر من المتبقي | صحّح المبلغ أو وزّع على فواتير |
| تكرار رمز (قديم) | تعارض نادر | اتصل بالدعم؛ التوليد الجديد يمنع التكرار |
| Posting Guard | فترة مغلقة أو قيد غير متوازن | راجع الفترة المحاسبية والبنود |
| فشل الترحيل | Rollback تلقائي للمعاملة | أعد المحاولة بعد تصحيح البيانات |

---

## 7. الأمان

- **كلمات المرور:** bcrypt؛ لا تُخزَّن ولا تُعاد في API.
- **الجلسات:** JWT + سجل جلسات مع جهاز وعنوان IP.
- **المسارات:** `/api/*` تتطلب مصادقة؛ العمليات الحساسة تتطلب صلاحية محددة.
- **حذف فاتورة / عكس دفع / تسوية مخزون:** محمية بصلاحيات `delete_*` / `reverse_*` / `adjust_inventory`.

---

## 8. قوة هذا النظام

- **مصدر حقيقة واحد** للمخزون والمالية.
- **سجل تدقيق** لكل إنشاء/تعديل/حذف.
- **محاسبة آلية** مرتبطة بالعمليات التشغيلية.
- **تزامن مالي ومخزني** بدون إدخال يدوي للقيود في المسار القياسي.
